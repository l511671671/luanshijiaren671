# 深度推理先规划再执行 — Fable 5 级推理深度

## 核心原则

**慢思考，快执行。** Fable 5 的"一次性干对"不是猜得准，是提前想清楚所有方案和边界。这条技能强制我在动手前做完脑力工作。

## 触发条件

满足以下**任意一条**就激活本技能：
- 用户问"怎么实现""用什么方案""选哪个"
- 涉及架构决策、技术选型、算法设计
- 问题有 2+ 种可行解法
- 改动影响 3+ 个文件

## 推理框架（5 步）

### 第 1 步：方案穷举
列出所有可行方案（最少 2 个，不要只有 1 个）：
- 方案 A：[名称] — [一句话思路]
- 方案 B：[名称] — [一句话思路]
- 方案 C（如存在）：[名称]

### 第 2 步：多维度对比

| 维度 | 方案 A | 方案 B | 方案 C |
|:--|:--|:--|:--|
| 实现复杂度 | | | |
| 性能影响 | | | |
| 维护成本 | | | |
| 边界安全性 | | | |
| 与现有系统兼容 | | | |

### 第 3 步：失败模式预判
对首选方案，列出最可能的 3 种失败模式：
1. [失败场景 1] → [概率] → [后果]
2. [失败场景 2] → [概率] → [后果]
3. [失败场景 3] → [概率] → [后果]

### 第 4 步：边界条件穷举
- 空值/零值/None
- 极大值/极小值
- 并发/竞态
- 网络断开
- 权限不足
- 编码问题（中文/Gbk）

### 第 5 步：决策输出
**明确推荐方案 + 理由**（不是"都可以"），然后执行。

## 输出格式

```
## 推理决策：[问题描述]

### 方案对比
[对比表]

### 推荐：方案 X
[核心理由，1-3 句话]

### 风险提示
- [主要风险] — [应对措施]

开始执行。
```

## 与 Self-Validate 的协作
方案执行完成后，self-validate 会校验输出是否与推理结论一致。
