# 自检验证 — Fable 5 级交付纪律

## 核心原则

**宁可多花 30% 时间验证，不可交付未验证的结果。** Fable 5 在"最高努力"模式下会反思和验证自己的工作——这条技能强制同样的纪律。

## 验证清单（逐项执行）

每完成一个任务，在报告"完成"之前，逐项确认：

### 1. 运行输出验证
- **代码**：跑一遍确保能启动/运行。如果写了测试，运行测试。
- **数据**：抽样核对 3-5 条结果，确保数字落在合理范围内。
- **配置**：检查 JSON 格式/缩进无误。

### 2. 回归检查
- 改动是否影响已有功能？
- 搜索 `from xxx import` 和 `import xxx`，确保依赖完整。
- 搜索被修改函数的引用处，确认接口兼容。

### 3. 目标校对
- 回头看用户原始需求，逐条勾对：
  - [ ] 需求 A 已满足
  - [ ] 需求 B 已满足
  - [ ] 没有遗漏项

### 4. 边缘情况
- 空输入/空列表/文件不存在时是否崩溃？
- 网络异常时是否有兜底？

### 5. （如有 UI/图表/可视化）视觉复核
- 读到输出文件，确认关键元素存在。
- 布局不跑偏，数据标签不溢出。

## 报告格式

验证通过后，报告使用以下格式：

```
✅ 自检通过
- [x] 运行输出验证
- [x] 回归检查
- [x] 目标校对
- [x] 边缘情况
```
