# 基金 / 基金业绩表现 / 基金份额与分红

## 基金分红

接口路径：`fund/dividend`
请求方式：**`POST`**
tool_id：`list_fund_dividend_distributions`

接口说明：通过基金代码和时间范围查询基金分红信息，包括分红年度、分红对象、单位基金收益、分红比例、分红总额等关键数据，适用于基金分红分析和投资决策。

### 输入参数

**Query 参数**

_无参数_

**Body JSON 参数**

| 参数名 | 必填 | 类型 | 说明 | 示例 |
|--------|:----:|------|------|------|
| `fundCode` | — | string | 基金代码【与fundCodes组成多选一参数，必须且只能传递其中一个】 | `000001` |
| `fundCodes` | — | array | 基金代码列表【与fundCode组成多选一参数，必须且只能传递其中一个】 | `['000001', '000006']` |
| `beginDate` | — | string | 开始日期（格式yyyy-MM-dd） | `2025-01-01` |
| `endDate` | — | string | 结束日期（格式yyyy-MM-dd） | `2025-01-01` |
| `pageNum` | — | integer | 页码。 最小值:1; | `1` |
| `pageSize` | — | integer | 页长。 最小值:1; 最大值:500; | `10` |

### 输出参数

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `date` | 发布日期(yyyy-MM-dd) | `2016-01-13` |
| `fundCode` | 基金代码 | `169102` |
| `fundId` | 今日投资内部编码 | `169102JK` |
| `fundName` | 基金名称 | `华夏沪深300ETF联接A` |
| `dividendYear` | 分红年度 | `2015` |
| `dividendTarget` | 分红对象 | `基金持有人` |
| `fundIncomePerUnit` | 单位基金收益(元) | `0.1234` |
| `fundUndistributedIncomePerUnit` | 单位基金未分配收益(元) | `0.0567` |
| `isDividend` | 是否分红 | `1` |
| `dividendRatioBeforeTax` | 分红比例(含税10派X元) | `2.95` |
| `dividendRatioAfterTax` | 分红比例(税后) | `2.95` |
| `totalDividendAmount` | 分红总额(元) | `1234567.89` |
| `recordDate` | 权益登记日(yyyy-MM-dd) | `2016-01-20` |
| `exDividendDate` | 除息日(yyyy-MM-dd) | `2016-01-20` |
| `dividendPayDate` | 分红发放日(yyyy-MM-dd) | `2016-01-22` |
| `planChangeDescription` | 方案变更说明 | `无变更` |

### 接口示例

```bash
# Body JSON 可选参数: fundCode, fundCodes, beginDate, endDate, pageNum, pageSize
investoday-api fund/dividend --method POST --body-json '{"fundCode":"000001","fundCodes":["000001","000006"],"beginDate":"2025-01-01"}'
```

## 基金拆分折算

接口路径：`funds/share-splits`
请求方式：**`POST`**
tool_id：`list_fund_share_splits`

接口说明：通过基金代码和日期范围（格式为yyyy-MM-dd HH:mm:ss）查询基金份额拆分与折算的历史记录，返回包括拆分日、拆分比例、拆分对象、拆分后净值、累计净值披露说明及拆分说明等核心信息，用于分析基金份额变动和净值调整情况。

### 输入参数

**Query 参数**

_无参数_

**Body JSON 参数**

| 参数名 | 必填 | 类型 | 说明 | 示例 |
|--------|:----:|------|------|------|
| `fundCode` | — | string | 基金代码【与fundCodes组成多选一参数，必须且只能传递其中一个】 | `000001` |
| `fundCodes` | — | array | 基金代码列表【与fundCode组成多选一参数，必须且只能传递其中一个】 | `['000001', '000006']` |
| `beginDate` | — | string | 开始日期（格式yyyy-MM-dd） | `2025-01-01` |
| `endDate` | — | string | 结束日期（格式yyyy-MM-dd） | `2025-01-01` |
| `pageNum` | — | integer | 页码。 最小值:1; | `1` |
| `pageSize` | — | integer | 页长。 最小值:1; 最大值:500; | `10` |

### 输出参数

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `fundCode` | 基金代码 | `000001` |
| `fundName` | 基金名称 | `华夏成长混合` |
| `fundId` | 今日投资内部编码 | `000006jk` |
| `splitExDate` | 拆分日 | `2024-08-15` |
| `splitRatio` | 拆分比例 | `0.75` |
| `splitTarget` | 拆分对象 | `订单号ORD20231027001` |
| `navAfterSplit` | 拆分后基金净值 | `1.0524` |
| `navAccDisclosureNote` | 拆分后累计净值披露说明 | `本基金于2023年6月15日进行份额拆分，比例为1:1.5。拆分后，基金份额净值调整为1.0000元，并以此为基础开始计算累计净值。拆分前的累计净值增长已按拆分比例折算并入当前累计净值中。` |
| `splitNote` | 拆分说明 | `因业务调整，原订单拆分为子订单A001和A002分别处理` |

### 接口示例

```bash
# Body JSON 可选参数: fundCode, fundCodes, beginDate, endDate, pageNum, pageSize
investoday-api funds/share-splits --method POST --body-json '{"fundCode":"000001","fundCodes":["000001","000006"],"beginDate":"2025-01-01"}'
```
