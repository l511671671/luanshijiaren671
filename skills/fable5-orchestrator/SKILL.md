# Fable 5 Orchestrator — 复杂任务全链路调度

## 作用

当你接到一个复杂任务时，本技能自动帮你：
1. **规划** → 拆分子任务
2. **推理** → 对比方案、预判风险
3. **执行** → 按步骤推进
4. **验证** → 每个子任务校验
5. **收尾** → 跨会话时写 checkpoint
6. **纠错** → 出错时按错误恢复流程处理

---

## 触发条件

满足任意一条即激活：
- 用户说 "做个方案""设计一下""怎么实现"
- 任务预计 3 次以上工具调用
- 涉及 2 个以上文件修改
- 任务可能跨会话
- 需要对比多个方案
- 涉及架构/技术选型

## 工作流程

```
启动
  ↓
读 fable5-framework/README.md 了解框架
  ↓
判断是否需要规划？
  ├─ 是 → 读 01-planning.md，生成 plan
  ↓
判断是否需要深度推理？
  ├─ 是 → 读 02-reasoning.md，做方案对比
  ↓
按 plan 逐步执行
  ├─ 每步参考 03-execution.md
  ├─ 每步完成后参考 04-verification.md 验证
  ↓
判断是否需要跨会话？
  ├─ 是 → 读 06-session-management.md，写 checkpoint
  ↓
最终按 07-quality-gates.md 验收
  ↓
交付
```

## 协作技能

| 技能 | 作用 | 调用时机 |
|------|------|----------|
| `deep-reason` | 深度推理 | 方案对比、架构决策 |
| `self-validate` | 自检验证 | 每个子任务完成后 |
| `long-horizon` | 断点续传 | 跨会话任务 |
| `fable5-orchestrator` | 整体调度 | 复杂任务启动时 |

---

## 纪律

1. 复杂任务不直接动手，先规划
2. 多方案必须先对比再决策
3. 每个子任务必须有验证
4. 长任务必须有 checkpoint
5. 最终交付必须通过质量门
