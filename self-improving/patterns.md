# WorkBuddy Patterns

可复用的工作模式与最佳实践。

## Pattern 1: 先验证，再交付

**场景**: 代码、数据、配置类任务
**步骤**:
1. 完成主要修改
2. 运行最小化验证（启动、抽样、回归）
3. 确认通过后才报告完成
**收益**: 避免返工，提升信任度

## Pattern 2: 数据源降级链

**场景**: 金融行情/交易数据
**优先级**:
1. 本地缓存 (`kline_cache.db`)
2. 腾讯财经 API
3. iFinD
4. 东方财富（限速使用）
**原则**: 高频调用优先走本地或腾讯，避免东财被封

## Pattern 3: 复杂任务先拆清单

**场景**: 多步骤方案/报告/营销计划
**步骤**:
1. 列出关键模块
2. 每个模块标注：输入、输出、截止时间、依赖
3. 先完成骨架，再填充细节

## Pattern 4: 跨会话续接

**场景**: 长周期任务
**步骤**:
1. 任务开始时评估是否需要 `LONG_RUN_CHECKPOINT.md`
2. 每阶段结束写入进度
3. 新会话先读取 checkpoint

## Pattern 5: 用户纠正即时归档

**场景**: 任何用户指出错误或偏好时
**步骤**:
1. 立即修正
2. 写入 `corrections.md` 或 `MEMORY.md`
3. 下次同场景自动应用
