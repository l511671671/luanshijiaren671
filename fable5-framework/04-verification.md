# 04 多层验证体系（Verification）

**目标：** 每个交付物在交给用户前都经过充分验证。

---

## 验证层级

### L1：运行时验证
- 代码能启动
- 没有语法错误
- 基本流程能跑通

### L2：数据正确性验证
- 抽样核对 3-5 条数据
- 指标口径一致
- 数值在合理范围内

### L3：回归验证
- 改动没有破坏已有功能
- 接口兼容性保持
- 依赖关系完整

### L4：目标对齐验证
- 回顾用户原始需求
- 检查是否有遗漏
- 确认输出格式符合预期

### L5：边界验证
- 空输入不崩溃
- 异常网络有兜底
- 大文件/大数据量能处理

---

## 验证清单模板

### 通用交付

```markdown
## 自检清单
- [ ] 运行时验证通过
- [ ] 数据抽样核对通过
- [ ] 回归检查通过
- [ ] 目标对齐检查通过
- [ ] 边界情况处理检查通过
```

### 代码交付

```markdown
- [ ] 能启动 / 能运行
- [ ] 测试通过（如有测试）
- [ ] 关键路径有日志
- [ ] 依赖已检查
- [ ] 没有硬编码敏感信息
- [ ] 中文路径/编码处理正确
```

### 数据交付

```markdown
- [ ] 数据来源已说明
- [ ] 抽样核对 3-5 条
- [ ] 缺失值/异常值已处理
- [ ] 字段口径一致
- [ ] 输出格式正确
```

### 文档交付

```markdown
- [ ] 结构清晰
- [ ] 数据有来源
- [ ] 建议可执行
- [ ] 无错别字
- [ ] 格式统一
```

---

## 视觉复核

对于 UI、图表、文档：
- [ ] 关键元素存在
- [ ] 布局不跑偏
- [ ] 数据标签不溢出
- [ ] 中文字体正常
- [ ] 表格边界对齐

---

## 验证原则

1. **宁可多花 30% 时间验证，也不交付未验证的结果**
2. **验证不是最后一步，而是每一步**
3. **不能自证的结论不交付**
4. **发现错误立即修正并记录**
